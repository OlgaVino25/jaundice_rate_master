# Фильтр желтушных новостей

Асинхронный анализатор новостных статей на предмет эмоционально окрашенной («жёлтой») лексики.  
Позволяет оценить долю «заряженных» слов в тексте, вычисляя *коэффициент желтушности*.  
Поддерживается работа с сайтом **ИноСМИ.ру** (через адаптер), код легко расширяется для других источников.

## Возможности

- CLI-утилита для анализа статей из командной строки
- Веб-сервер с REST API (JSON-ответы)
- Асинхронное скачивание и обработка нескольких статей одновременно
- Автоматические повторные попытки при сетевых ошибках
- Таймауты на скачивание и анализ (настраиваемые)
- Логирование всех этапов работы
- Подготовленная архитектура для добавления новых источников (адаптеры)

## Технологии

- Python 3.11+
- **aiohttp** – асинхронные HTTP-запросы
- **pymorphy2** – морфологический анализ (лемматизация)
- **BeautifulSoup4** – парсинг HTML
- **anyio** – управление конкурентными задачами (в CLI-версии)
- **pytest** – тестирование

# Запуск CLI-анализатора (main.py)
Скрипт принимает один или несколько URL статей и выводит результаты в консоль.

```bash
python main.py <url1> [url2 ...] [--options]
```

**Параметры**

Параметр | По умолчанию | Описание
---------|--------------|----------
`--fetch-timeout` | 5 | Таймаут скачивания страницы (сек)
`--parse-timeout` | 3 | Таймаут морфологического анализа (сек)
`--retry-attempts` | 3 | Количество попыток при сетевых ошибках
`--retry-delay` | 10 | Начальная задержка между попытками (сек)

**Пример**

```bash
python main.py https://inosmi.ru/20260610/iran-278840067.html https://inosmi.ru/20260609/ukraina-278825492.html --fetch-timeout 10
```

## Запуск веб-сервера (server.py)

Сервер принимает GET-запросы по адресу http://127.0.0.1:8080/ с параметром urls (список URL через запятую) и возвращает JSON.

```bash
python server.py
```

**Пример запроса**

```bash
curl "http://127.0.0.1:8080/?urls=https://inosmi.ru/20260609/ukraina-278825492.html,https://inosmi.ru/20260610/iran-278840067.html"
```

**Пример ответа**

```json
[
  {
    "url": "https://inosmi.ru/20260609/ukraina-278825492.html",
    "status": "OK",
    "score": 1.05,
    "words_count": 477,
    "time": 0.45
  },
  {
    "url": "https://inosmi.ru/20260610/iran-278840067.html",
    "status": "OK",
    "score": 0.92,
    "words_count": 764,
    "time": 0.44
  }
]
```
Результат обработки
**Поля ответа:**

Поле | Тип | Описание
-----|-----|------------
`url` | string | Адрес статьи
`status` | `OK` / `FETCH_ERROR` / `PARSING_ERROR` / `TIMEOUT` | Результат обработки
`score` | float или `null` | Доля заряженных слов в процентах (0–100)
`words_count` | int или `null` | Количество слов в очищенном тексте
`time` | float или `null` | Время обработки статьи (сек)

Сервер ограничивает количество URL в одном запросе – не более 10 (константа MAX_URLS).
При превышении возвращается ошибка 400.

## Архитектура проекта

```text
.
├── adapters/               # Адаптеры для разных источников новостей
│   ├── exceptions.py       # Кастомные исключения (ArticleNotFound)
│   ├── html_tools.py       # Утилиты для очистки HTML
│   └── inosmi_ru.py        # Парсер и очистка статей ИноСМИ
├── charged_dict/           # Словари «заряженных» слов (negative_words.txt, positive_words.txt)
├── text_tools.py           # Морфологический анализ, расчёт желтушности
├── main.py                 # CLI-клиент
├── server.py               # Веб-сервер (aiohttp)
├── tests/                  # Модульные и интеграционные тесты
└── requirements.txt        # Зависимости
```

**Ключевые компоненты:**

- **Адаптеры** – изолируют логику извлечения текста из HTML конкретного сайта.
- **text_tools** – чистые функции для нормализации слов и вычисления рейтинга.
- **Сетевая часть** – повторные попытки, таймауты, асинхронный конкурентный запуск (через asyncio.gather в сервере и anyio.create_task_group в CLI).

## Дополнительные настройки

Все числовые параметры вынесены в константы в начале файлов (`server.py`, `main.py`):

- `REQUEST_TIMEOUT` – таймаут скачивания страницы
- `ANALYSIS_TIMEOUT` – таймаут лемматизации
- `MAX_URLS` – максимальное число URL в одном запросе к серверу
- `RETRY_ATTEMPTS`, `RETRY_DELAY` – управление повторными попытками

Вы можете изменить их в коде или через аргументы командной строки (для CLI).
